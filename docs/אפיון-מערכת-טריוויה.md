# מסמך אפיון - מערכת טריוויה טלפונית

## סקירה כללית

מערכת חידון טריוויה אינטראקטיבית שבה משתתפים מחייגים למספר טלפון ולוחצים 1-4 כדי לענות על שאלות. האדמין מנהל את המשחק דרך ממשק ווב, והמסך הציבורי מוקרן לכולם.

---

## משתמשים במערכת

| סוג | תיאור |
|-----|--------|
| אדמין | מנהל החידון - נכנס עם שם משתמש וסיסמה |
| משתתף | מחייג למספר טלפון, לוחץ 1-4 לתשובות |
| צופה | רואה את המסך הציבורי בלבד |

מספר משתתפים: 1-20 (ניתן לשינוי בקוד)

---

## זרימת מסכים

### צד האדמין

```
התחברות → רשימת חידונים → עריכת חידון → לובי → לוח בקרה → דוח סיום
```

**1. התחברות** (`/login`)
- שם משתמש + סיסמה
- JWT token נשמר ב-localStorage

**2. רשימת חידונים** (`/quizzes`)
- רשימת כל החידונים עם סטטוס (טיוטה / פעיל / הסתיים)
- יצירת חידון חדש (שם + תיאור)
- כפתורים: ערוך / לוח בקרה / דוח

**3. עריכת חידון** (`/quiz/:id/edit`)
- טאב שאלות: הוספה ידנית / ייבוא אקסל / בחירה ממאגר
- טאב משתתפים: הוספה ידנית / ייבוא אקסל / קישור הרשמה עצמית
- טאב הגדרות: זמן לשאלה, רקע, מנגינה, הגדרות ניקוד
- כפתור "התחל חידון" → עובר ללובי

**4. לובי** (`/quiz/:id/lobby`)
- מסך המתנה לפני תחילת המשחק
- מציג מספר טלפון גדול לחיוג
- מונה משתתפים מחוברים בזמן אמת
- כפתור "התחל עכשיו" → עובר ללוח בקרה

**5. לוח בקרה** (`/dashboard/:id`)
- רשימת שאלות עם כפתור הפעלה לכל שאלה
- שאלה פעילה: תצוגת תוצאות בזמן אמת, טיימר, כפתורי עצור/חשוף
- לוח מובילים בצד
- כפתור "מסך ציבורי" לפתיחה בחלון נפרד
- כפתור "סיים חידון"

**6. דוח סיום** (`/reports/:id`)
- סטטיסטיקות לכל שאלה
- לוח מובילים סופי

---

### צד המסך הציבורי (`/display/:id`)

```
המתנה → שאלה נכנסת → תשובות נכנסות → טיימר רץ + שמות עונים → עצירה + סטטיסטיקות → חשיפת תשובה + מי הכי מהיר → [לוח מובילים] → מסך מנצח
```

**שלבי הצגת שאלה:**
1. טקסט השאלה מגלש פנימה (אנימציה 1 שניה)
2. כל תשובה מופיעה בנפרד עם אנימציה (כל 0.5 שניה)
3. טיימר מתחיל לרוץ (בר צבעוני שמתקצר, מאיץ ב-3 שניות אחרונות עם צליל)
4. **בזמן הטיימר: כל פעם שמישהו עונה - שמו עולה על המסך** ("⚡ ישראל ישראלי ענה!")
5. אחרי הטיימר: בר אחוזים לכל תשובה
6. חשיפת תשובה נכונה: ירוק + "✓ נכון!", שאר מתעממות
7. **מציג: מי היה הכי מהיר** (שם + זמן תגובה)
8. מציג: כמה ענו נכון, כמה ענו בסך הכל

**לוח מובילים** - מוצג כל X שאלות (ניתן להגדרה, ברירת מחדל: 5)

**מסך מנצח** - בסוף החידון: שם המנצח + ניקוד + קונפטי

---

### צד המשתתף (טלפון)

```
חיוג → ברכה קולית → ממתין לשאלה → שומע שאלה → לוחץ 1-4 → שומע אישור → ממתין לשאלה הבאה
```

- אפשרות לא להצביע - המשתתף פשוט לא לוחץ כלום, לא עולה ולא יורד לו

## סוגי שאלות

### 1. שאלה רגילה
- 2-4 תשובות אפשריות
- תשובה אחת נכונה
- ניקוד מבוסס מהירות

### 2. שאלת בונוס
- מסומנת בתג מיוחד על המסך
- ניקוד מוכפל x2
- המנהל יכול להפעיל/לכבות בהגדרות
- מופיעה מדי פעם (לא כל שאלה)
- אופציונלית - מי שרוצה משתתף

### 3. שאלת סקריש (Scratch)
- השאלה מתגלה לאט - אות אחרי אות
- ניתן לענות בכל שלב
- מי שמנחש מוקדם יותר מקבל יותר נקודות

### 4. שאלת סקר
- ללא תשובה נכונה
- רק רואים את התפלגות התשובות
- ללא ניקוד
- מתאים לפתיחת ערב או שאלות כיפיות

### 5. שאלה עם תמונה
- תמונה מוצגת מעל השאלה
- "מה החיה הזו?" / "מה המקום הזה?"
- ניתן להעלות תמונה בעת יצירת השאלה

## מערכת ניקוד

### ניקוד בסיסי
- תשובה נכונה: **150 נקודות בסיס**
- מינוס **10 נקודות לכל שניה** שעברה
- מינימום: **50 נקודות** לתשובה נכונה
- תשובה שגויה: **0 נקודות**
- לא ענה: **0 נקודות** (לא מפסיד)

**דוגמאות (טיימר 10 שניות):**
- ענה ב-0 שניות = 150 נקודות
- ענה ב-5 שניות = 100 נקודות (ממוצע)
- ענה ב-10 שניות = 50 נקודות

**חידון 50 שאלות → ממוצע ~5,000 נקודות למנצח**

### ניקוד יחסי חכם (ניתן להפעלה/כיבוי על ידי המנהל)
- אם רק משתתף אחד ענה נכון → בונוס 50%
- אם פחות מ-30% ענו נכון → בונוס 25% לכולם שענו נכון
- אם אף אחד לא ענה נכון → השאלה הבאה קלה יותר (אם יש)

### שאלת בונוס
- כל הניקוד מוכפל x2

## תצוגת שמות בזמן אמת

כשמשתתף עונה במהלך הטיימר:
- שמו מופיע על המסך עם אנימציה ("⚡ ישראל ישראלי ענה!")
- נשאר על המסך כמה שניות ואז נעלם
- לא מגלה איזו תשובה בחר

אחרי חשיפת התשובה:
- מוצג "🏆 הכי מהיר: [שם] - [זמן] שניות"

## מאגר שאלות

### נושאים (מתאים לציבור החרדי)
- גיאוגרפיה (ארצות, ערים, נהרות)
- חיות וצמחים
- מדע כללי
- היסטוריה כללית
- מספרים וחשבון
- כללי ידע

### ייבוא שאלות מאקסל
עמודות:
```
שאלה | תשובה נכונה | תשובה 2 | תשובה 3 | תשובה 4 | נושא | סוג (רגיל/בונוס/סקר/סקריש) | קושי (1/2/3) | תמונה (URL)
```
- התשובה הראשונה היא תמיד הנכונה
- המערכת מערבבת את סדר התשובות בעת הצגה

### ייבוא משתתפים מאקסל
עמודות:
```
שם | טלפון
```
- ניתן להעלות קובץ אקסל עם רשימת משתתפים בטאב "משתתפים"

## מבנה מסד הנתונים

### טבלאות קיימות
```sql
users               -- אדמינים
questions           -- שאלות (text, options JSONB, correct_answer, type, difficulty, image_url)
quizzes             -- חידונים (name, status, settings JSONB)
quiz_questions      -- קשר חידון-שאלות עם סדר
participants        -- משתתפים רשומים (name, phone, quiz_id)
participant_scores  -- ניקוד לכל שאלה (כולל זמן תגובה, is_fastest)
quiz_reports        -- דוחות סיום
call_logs           -- לוג שיחות טלפון
```

### טבלאות לעתיד
```sql
question_categories  -- נושאים למאגר
quiz_settings        -- הגדרות מפורטות (זמן, רקע, מנגינה, ניקוד יחסי)
backgrounds          -- רשימת רקעים זמינים
music_tracks         -- רשימת מנגינות זמינות
```

### שדות חשובים ב-participant_scores
```sql
answered_at_ms   -- כמה מילישניות לקח לענות
is_fastest       -- האם היה הכי מהיר בשאלה זו
bonus_multiplier -- מכפיל בונוס (1 רגיל, 2 בונוס)
```

## ארכיטקטורה טכנית

| רכיב | טכנולוגיה |
|------|-----------|
| Frontend | React + TypeScript |
| Backend | Node.js + Express + TypeScript |
| DB | PostgreSQL (Clever Cloud) |
| Real-time | Socket.io |
| State | In-memory (ללא Redis) |
| Auth | JWT |
| Hosting | Railway |
| טלפוניה | ימות המשיח / Twilio (עתידי) |

### WebSocket events
```
quiz:question-activated   → שאלה חדשה התחילה
quiz:timer-tick           → עדכון טיימר כל שניה
quiz:participant-answered → משתתף ענה (שם בלבד, לא התשובה)
quiz:results-update       → עדכון תוצאות בזמן אמת
quiz:question-stopped     → הטיימר נעצר
quiz:answer-revealed      → תשובה נחשפה + מי הכי מהיר
quiz:leaderboard          → הצגת לוח מובילים
quiz:ended                → החידון הסתיים + מנצח
quiz:participant-count    → עדכון מספר מחוברים
```
